# Mission: Become a Complete CTO — Technical Mastery + Security + ML + Communication

## Why
Tom is already functioning as CTO at applyGOAT but recognises the gap between execution and genuine mastery. He wants to build five compounding skill pillars over time: architect-level system thinking, senior-grade coding craft, deep problem-solving instincts, cybersecurity expertise, machine learning depth, and the communication skills to lead, pitch, and influence at every level — engineer, investor, and board.

## Success looks like
- Designs any non-trivial system from first principles: trade-offs articulated, failure modes named, scaling path clear
- Writes production-quality code at senior level: clean, performant, testable, idiomatic Python and TypeScript
- Solves hard algorithmic problems with structured methodology under pressure
- Understands offensive and defensive security well enough to architect secure systems AND audit them
- Builds and understands ML/AI systems beyond "use the API" — knows the math, the architecture, the production concerns
- Communicates with precision and authority to engineers, investors, clients, and peers
- Passes senior/staff-level system design AND coding interviews at top-tier companies

## Pillars (teaching is organised around these)

### 1. Architect Knowledge
- Distributed systems fundamentals (CAP, consistency models, replication)
- System design patterns (caching, queues, sharding, API gateways, CDNs)
- Database internals and design (indexing, ACID vs BASE, schema design)
- Microservices & event-driven architecture (CQRS, event sourcing, service mesh)
- Cloud architecture on Azure (Container Apps, Functions, multi-tenant SaaS patterns)
- AI/ML system design (RAG at scale, model serving, agentic architecture)

### 2. Senior Coding Craft
- Data structures & algorithms — for interviews AND for writing better systems
- Design patterns (GoF, architectural patterns: repository, CQRS, saga)
- SOLID principles, clean code, idiomatic Python/TypeScript
- Performance: profiling, async patterns, caching, query optimisation
- Testing discipline: TDD, unit/integration/e2e, mocking strategy
- API design: REST maturity, versioning, contract-first design

### 3. Deep Problem Solving
- Structured decomposition methodology
- Trade-off analysis framework (time vs space, consistency vs availability, build vs buy)
- Debugging and root-cause methodology
- Technical decision-making under uncertainty

### 4. Cybersecurity
- Security fundamentals: CIA triad, threat modelling, attack surfaces
- Web security: OWASP Top 10, injection, XSS, CSRF, auth flaws
- Network security: TLS, firewalls, VPNs, zero-trust architecture
- Application security: secrets management, JWT hardening, OAuth2 security, RBAC at scale
- Cloud security on Azure: IAM, key vaults, network isolation, compliance
- AI security: prompt injection, LLM jailbreaks, model poisoning, API key governance (KeyShield relevance)
- Offensive basics: understanding how attacks work to defend better (CTF-level)

### 5. Machine Learning
- ML fundamentals: supervised/unsupervised/reinforcement, loss functions, gradient descent
- Deep learning: neural networks, transformers, attention mechanisms
- NLP and LLMs: tokenisation, embeddings, fine-tuning, RLHF, prompt engineering at depth
- RAG architecture: chunking, retrieval strategies, reranking, evaluation (applyGOAT anchor)
- ML in production: model serving, monitoring, drift detection, MLOps
- Vector databases: Pinecone internals, ANN algorithms (HNSW, IVF), similarity search
- AI agents: agentic architectures, LangGraph internals, tool use, memory systems

### 6. Communication
- Technical writing: clear documentation, ADRs, design docs, SRS (Mini Hermes anchor)
- Engineering leadership communication: running design reviews, giving feedback, unblocking teams
- Founder communication: investor updates, pitching technical vision, board-level clarity
- Stakeholder management: translating technical decisions into business impact
- Public speaking and presenting: demos, conference talks, team presentations
- Writing precision: being brief without losing depth (the CTO superpower)

## Constraints
- Tom has a running product and company — sessions must be high-density, no wasted time
- All examples must anchor to his real stack and projects
- Pacing is fast — skip what he clearly already knows in production

## Out of scope
- Beginner-level content in languages/frameworks he already uses in production
- Non-technical career coaching unrelated to his technical leadership growth
