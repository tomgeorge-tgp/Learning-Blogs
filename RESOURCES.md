# Technical Leadership & Senior Engineering Resources

## Knowledge

### System Design
- [System Design Primer — Donne Martin (GitHub)](https://github.com/donnemartin/system-design-primer)
  Comprehensive, well-maintained reference. Use for: any system design interview topic — caching, load balancing, databases, microservices.

- [Designing Data-Intensive Applications — Martin Kleppmann](https://dataintensive.net/)
  The canonical book on distributed systems, databases, and data engineering. Use for: deep dives on consistency, replication, event sourcing, stream processing.

### AI / RAG / Agentic Systems
- [LangGraph Documentation](https://langchain-ai.github.io/langgraph/)
  Official docs. Use for: agent loops, state machines, checkpointing, multi-agent patterns.

- [RAG Survey — Gao et al. (arXiv 2312.10997)](https://arxiv.org/abs/2312.10997)
  Peer-reviewed survey of RAG architectures. Use for: naive vs advanced RAG, chunking strategies, retrieval evaluation.

### Auth & Security
- [OAuth 2.0 RFC 6749](https://www.rfc-editor.org/rfc/rfc6749)
  Primary spec. Use for: authoritative OAuth2 flow definitions.

- [JWT.io Introduction](https://jwt.io/introduction)
  Concise, high-trust intro. Use for: JWT structure, signing, validation.

### DevOps / Docker
- [Docker Multi-stage Build Docs](https://docs.docker.com/build/building/multi-stage/)
  Official. Use for: multi-stage build patterns.

- [Caddy Documentation](https://caddyserver.com/docs/)
  Official. Use for: reverse proxy config, automatic TLS, Caddyfile syntax.

### Cloud / Azure
- [Azure Architecture Center](https://learn.microsoft.com/en-us/azure/architecture/)
  Microsoft's reference architectures. Use for: Container Apps, Functions, multi-tenant SaaS patterns on Azure.

## Wisdom (Communities)

- [r/ExperiencedDevs](https://reddit.com/r/ExperiencedDevs)
  High signal-to-noise for senior/staff-level discussions. Use for: career growth, architectural debates, interview prep at senior level.

- [Hacker News](https://news.ycombinator.com)
  Use for: staying current on AI/agentic tooling, distributed systems discussions, founder-level technical conversations.

- [MLOps Community Slack](https://mlops.community/)
  Use for: RAG, LLM production, agentic system discussions with practitioners.

## Gaps
- No high-trust resource yet for MCP (Model Context Protocol) — official Anthropic docs are the best current source but evolving fast.
- No resource yet for multi-tenant SaaS architecture patterns specific to Python/FastAPI.
